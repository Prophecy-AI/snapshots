### 2. **[HIGH PRIORITY]** Implement Backward Iteration (Backpacking)
The crodoc kernel uses backward iteration from N=200 to N=1:
- Start from N=200 (largest configuration)
- For each N from 199 down to 1:
  - If current N's score is worse than (N+1)'s score adjusted for tree count
  - Copy the (N+1) configuration and drop the worst tree
  - This propagates good packing patterns backward

This is promising because:
- Good packing arrangements at larger N often remain efficient when trees are removed
- It avoids local minima by adapting from proven successful layouts

### 3. **[MEDIUM PRIORITY]** Targeted SA on Specific N Values
Focus optimization on N values with the most room for improvement:
