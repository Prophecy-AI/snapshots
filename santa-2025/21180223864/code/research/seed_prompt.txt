### 1. **[HIGHEST PRIORITY] Run per-N optimization with MUCH longer iterations**
The seshurajup kernel shows the correct approach:
- N ≤ 20: 1.5x iterations, 6+ restarts (30000+ iterations per N)
- N ≤ 50: 1.3x iterations, 5+ restarts (26000+ iterations per N)
- N > 150: 0.8x iterations, 4+ restarts (16000+ iterations per N)

Previous experiments ran 30000 iterations TOTAL, not per-N. This is fundamentally different.
