## Recommended Approaches (Priority Order)

### 1. **[HIGHEST PRIORITY]** NFP-Based Constructive Heuristic
**Rationale**: The web research reveals that No-Fit Polygon (NFP) based constructive heuristics can find fundamentally different solutions than SA optimization. The NFP encodes all valid relative positions between two polygons.

**Implementation**:
1. Compute NFP for the Christmas tree polygon with itself at various angles
2. Use a greedy constructive heuristic: place trees one by one at the "best" NFP position
3. Try different placement orders (largest-first, random, etc.)
4. This generates solutions from a DIFFERENT starting topology

**Why this might work**: Our current solutions were optimized from a specific starting configuration. NFP-based construction explores a completely different solution space.

### 2. **[HIGH PRIORITY]** Partition-and-Sequential Placement (CG:SHOP 2024 approach)
**Rationale**: The CG:SHOP 2024 challenge winner used this approach for polygon packing:
1. Partition the container into small regions
2. Assign polygons to regions randomly
3. Solve each sub-problem with sequential NFP-based placement
4. This scales to 50,000 polygons and finds high-quality solutions

**For Santa 2025**: For each N, treat the bounding box as the container and use this approach.

### 3. **[MEDIUM PRIORITY]** Mathematical Exhaustive Search for Small N
**Rationale**: For N=2, 3, 4, the search space is small enough to explore exhaustively.

**Implementation for N=2**:
1. For θ from 0° to 180° in 0.01° increments (18,000 angle pairs)
2. For each angle pair (θ, θ+180°), compute optimal positions analytically
3. Find the angle pair that minimizes the bounding box
4. Compare with current N=2 solution (score=0.450779)

### 4. **[MEDIUM PRIORITY]** Double-Lattice Construction (Kuperberg)
**Rationale**: Web research mentions that dense 2-tree lattices with alternating up/down trees achieve good scores. The Kuperberg double-lattice construction finds optimal inscribed parallelograms.

**Implementation**:
1. Compute the optimal inscribed parallelogram for the Christmas tree polygon
2. Use the parallelogram's edges as lattice vectors
3. Place trees on two interleaved lattices (one up, one down)
4. This is a CONSTRUCTIVE approach, not optimization

### 5. **[LOW PRIORITY - ONLY IF OTHERS FAIL]** GFPack++ Style Learning
**Rationale**: GFPack++ uses neural networks to learn gradient fields for polygon placement. This is complex to implement but represents a fundamentally different approach.

**Note**: This requires training data (teacher packings) which we may not have.

## Key Technical Details for Implementation

### Christmas Tree Polygon (15 vertices):
```
TX = [0, 0.125, 0.0625, 0.2, 0.1, 0.35, 0.075, 0.075, -0.075, -0.075, -0.35, -0.1, -0.2, -0.0625, -0.125]
TY = [0.8, 0.5, 0.5, 0.25, 0.25, 0, 0, -0.2, -0.2, 0, 0, 0.25, 0.25, 0.5, 0.5]
```

### NFP Computation:
The No-Fit Polygon of polygon A with polygon B is computed by:
1. Reflect B about the origin to get -B
2. Compute the Minkowski sum of A and -B
3. The result is the NFP - its boundary represents all positions where A and B touch without overlapping

### Scoring:
```
score = sum over N from 1 to 200 of (side_length^2 / N)
```
where side_length is max(xmax-xmin, ymax-ymin) of the bounding box.
