### RECOMMENDED APPROACH: GENETIC ALGORITHM WITH TOPOLOGY CROSSOVER

**Why GA instead of CP:**
1. CP requires discretization which loses precision
2. GA can explore the continuous space more effectively
3. GA can exchange TREE ARRANGEMENTS between solutions (topology crossover)
4. GA has been successful in similar packing problems

**Key Innovation: TOPOLOGY CROSSOVER**
- Current approaches only optimize positions/angles within a fixed arrangement
- GA can exchange HOW trees are arranged (which tree is where)
- Crossover: Take tree positions from parent A, angles from parent B
- Mutation: Swap two trees, rotate subset, shift subset

**Implementation:**
1. Initialize population with variations of current best solution
2. Fitness = negative score (minimize bounding box)
3. Selection: Tournament selection
4. Crossover: 
   - Position crossover: Take positions from parent A, angles from parent B
   - Topology crossover: Swap tree assignments between parents
5. Mutation:
   - Small position/angle perturbation
   - Swap two trees
   - Rotate a subset of trees
6. Run for many generations with large population

### ALTERNATIVE: BRANCH AND BOUND FOR SMALL N

For N=2-5 specifically:
- These have the worst efficiency (1.7-1.8x theoretical)
- Use branch and bound to find provably optimal solutions
- If current solutions are optimal, B&B will confirm it
- If not, we get guaranteed improvements

### ALTERNATIVE: SIMULATED ANNEALING WITH TOPOLOGY MOVES

Instead of just position/angle moves, add:
- Swap two trees (exchange positions and angles)
- Rotate a subset of trees around their centroid
- Mirror a subset of trees
